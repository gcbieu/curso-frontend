1. Cores e Fundos

O Bootstrap tem uma paleta de 8 cores semânticas. Cada cor tem um significado — primary é a cor principal da marca, danger é para erros, success para confirmações etc. As mesmas cores se aplicam em texto (text-*) e em fundo (bg-*).

1.1 Cores de Texto — text-*

text-primary — Azul principal (identidade da marca)

text-secondary — Cinza (ação secundária)

text-success — Verde (confirmação, resultado positivo)

text-danger — Vermelho (erro, perigo, ação destrutiva)

text-warning — Amarelo (atenção, aviso moderado)

text-info — Azul claro (informação neutra)

text-muted — Cinza suave (texto secundário, legendas)

1.2 Cores de Fundo — bg-*
bg-primary + text-white
bg-secondary + text-white
bg-success + text-white
bg-danger + text-white
bg-warning (fundo claro — não usa text-white)
bg-light + border (quase branco, use borda para visualizar)
bg-dark + text-white
1.3 Personalizando com CSS Próprio

O Bootstrap 5 usa variáveis CSS (--bs-primary, --bs-danger, etc.) internamente em vez de valores fixos. Isso permite trocar toda a paleta do site sobrescrevendo apenas as variáveis no :root do seu CSS, sem precisar modificar ou baixar o Bootstrap: 

/* No seu style.css — DEPOIS do link do Bootstrap no head */
:root {
  --bs-primary:     #6C63FF;        /* substitui o azul padrão */
  --bs-primary-rgb: 108, 99, 255;   /* versão RGB para transparências */
}
💡 As cores desta página já foram personalizadas assim — o azul aqui é #004a99 (cor da InovaWeb), não o #0d6efd padrão do Bootstrap. Inspecione qualquer elemento azul no DevTools para confirmar!

3.3 Alert Dispensável — com botão de fechar
Formulário enviado! Entraremos em contato em breve.
🔌 JavaScript em ação! O atributo data-bs-dismiss="alert" é lido pelo JavaScript do Bootstrap. Quando o botão é clicado, o JS: 1) Remove a classe show do alert (inicia o fade out) 2) Aguarda a transição CSS terminar (300ms) 3) Remove o elemento do DOM completamente Você não escreveu uma linha de JavaScript — mas JS rodou. 6. Modal

Janela de diálogo que aparece sobre o conteúdo da página. O botão se conecta ao modal pelo ID via data-bs-toggle e data-bs-target.

6.1 Botões que abrem os Modals
🔌 JavaScript em ação! Quando o botão é clicado, o JS do Bootstrap executa em sequência: 1) Localiza o elemento com o ID definido em data-bs-target 2) Adiciona a classe show ao modal (tornando-o visível) 3) Cria e insere um backdrop escuro no DOM 4) Aplica overflow: hidden no body (bloqueia scroll) 5) Gerencia o foco do teclado e acessibilidade via aria Ao fechar: o processo é revertido com animação de fade (300ms).
